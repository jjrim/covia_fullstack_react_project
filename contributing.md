# Contributing

Thank you for your interest in contributing to our COVIA App. Your time and effort will be a great help to us.
Let’s build and develop an awesome app!

When you contribute to our repository, please let our team members know what change you wish to make via the group chat in Slack or Discord, so we can discuss about it together and bring up better ideas. You can find the detailed team information in README.md.


# Code of Conduct

## Standards

* Organizing the code efficiently
* Respecting team members’ different point of view
* Accepting team opinion and decision
* Avoiding unethical manner 


# To get started…

Please abide by the following steps:

1.	Look through the entire codes that our team has already created and ask team members if you have any questions through Slack or Discord.
2.	Bring up any brilliant ideas and discuss with team members in the meeting.
3.	After team agreement, you can clone our repository to your desktop and make a new local feature branch from Dev branch of the repository(Please ensure your new branch is derived from dev branch, not master). Dev branch has the latest changes from each members’ branches.
4.	Branch naming convention: firstName_lastName_featureName (ex. Alex_Smalltown_feature1).
5.	Work on your local feature branch.
6.	Commit and Push your work on your local feature branch.
7.	Submit a pull request from your local branch to the dev branch.
8.	After one of the team members merge your pull request to Dev branch, work on the next step or modify your code with any feedback.


>**Note**: 
>* Please make sure there are no errors on your code before commit and push.
>* Please let your team members know you have pushed your code so that they can review and merge it.
>* Please ensure there are no conflicts when you submit the pull request to the dev branch.
